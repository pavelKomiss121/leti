# Занятие 3. NumPy arrays and functions

## Краткая сводка (без кода)

**Массивы NumPy.** Основной объект — `numpy.ndarray`: однородный многомерный массив одного типа. Создание: `np.array(...)`, `np.zeros(shape)`, `np.ones(shape)`, для случайных — `np.random.rand(...)` (равномерное на [0,1)), `np.random.randn(...)` (нормальное распределение).

**Атрибуты ndarray.** `ndim` — число осей, `shape` — размер по каждой оси (кортеж), `size` — общее число элементов, `dtype` — тип элементов, `itemsize` — размер одного элемента в байтах.

**Функции в Python.** Определяются через `def имя(параметры):`, тело с отступом, `return` для возврата значения. Функция выполняется только при вызове.

**Типы колонок таблицы.** Каждую колонку можно представить как массив NumPy (например, `df[col].to_numpy()`) и посмотреть `dtype` — так получают тип данных «средствами NumPy». Результат можно записать в CSV (имя колонки, тип).

**Кодирование категорий.** Качественные признаки (Yes/No, Male/Female, Positive/Negative) для многих алгоритмов машинного обучения заменяют числами (0, 1, 2, …). После замены таблица содержит только числа; снова определяют типы колонок и записывают их в CSV — все колонки будут числовыми.

## Суть для защиты

- Разница между `np.random.rand` (равномерное) и `np.random.randn` (нормальное).
- Назначение атрибутов shape, ndim, dtype, size.
- Зачем переводить категории в числа: алгоритмы часто требуют числовой ввод; кодирование сохраняет структуру таблицы и позволяет однозначно восстановить соответствие (0 → первый класс, 1 → второй и т.д.).
